Раздел 14. Анимации и переходы

Учимся добавлять плавные переходы для блоков, списков и экранов, не ломая UX и производительность.

Цель урока

Освоить ключевые идеи раздела и закрепить их через короткую пошаговую практику.

Теория простыми словами

1) <Transition>

Используется для анимации появления/исчезновения одного элемента или компонента.

2) <TransitionGroup>

Подходит для списков: когда элементы добавляются, удаляются или меняют позицию.

3) CSS-классы переходов

Vue автоматически добавляет классы вида *-enter-from, *-enter-active, *-leave-to.

4) JavaScript hooks в переходах

Если CSS недостаточно, можно управлять анимацией через хуки: @before-enter, @enter, @leave.

5) Доступность и «не переанимировать»

Анимации должны помогать, а не мешать. Слишком долгие и резкие эффекты ухудшают UX.

Практика по шагам

Практическое задание: анимированный список задач и раскрывающиеся блоки.

Мостик: идем от базового появления элемента к анимации списков и потом к более управляемым сценариям через hooks.

Шаг 1. Появление/скрытие блока через <Transition>

<button @click="isOpen = !isOpen">Показать детали</button>

<Transition name="fade">
  <div v-if="isOpen">Раскрывающийся блок</div>
</Transition>
.fade-enter-active, .fade-leave-active {
  transition: opacity 0.25s ease;
}
.fade-enter-from, .fade-leave-to {
  opacity: 0;
}

Шаг 2. Анимированный список через <TransitionGroup>

<TransitionGroup name="list" tag="ul">
  <li v-for="task in tasks" :key="task.id">
    {{ task.title }}
  </li>
</TransitionGroup>
.list-enter-active, .list-leave-active {
  transition: all 0.2s ease;
}
.list-enter-from, .list-leave-to {
  opacity: 0;
  transform: translateY(8px);
}

Шаг 3. Переходы маршрутов (базовый пример)

<RouterView v-slot="{ Component }">
  <Transition name="page" mode="out-in">
    <component :is="Component" />
  </Transition>
</RouterView>

Шаг 4. JavaScript hooks в переходе

<Transition
  @before-enter="beforeEnter"
  @enter="enter"
  @leave="leave"
>
  <div v-if="visible">Блок с JS-анимацией</div>
</Transition>
const beforeEnter = (el) => { el.style.opacity = 0 }
const enter = (el, done) => {
  el.style.transition = 'opacity .2s'
  el.style.opacity = 1
  setTimeout(done, 200)
}
const leave = (el, done) => {
  el.style.transition = 'opacity .2s'
  el.style.opacity = 0
  setTimeout(done, 200)
}

Шаг 5. UX-правило: короткие и понятные переходы

/* Хорошая практика: 150-300ms */
.ui-enter-active, .ui-leave-active {
  transition: all .2s ease;
}

/* Учитывай пользователей с reduced motion */
@media (prefers-reduced-motion: reduce) {
  .ui-enter-active, .ui-leave-active {
    transition: none;
  }
}

Практическое задание

Собери анимированный список задач и раскрывающиеся блоки.

  1. Сделай раскрытие/скрытие блока через <Transition>.
  2. Сделай список задач через <TransitionGroup>.
  3. Добавь удаление задач с плавным уходом.
  4. Добавь page-transition для RouterView.
  5. Проверь, что анимация не мешает нажатию кнопок.

Мини-практика: toast-уведомления с короткой анимацией появления и исчезновения.

Типичные ошибки

  • Добавлять слишком долгие анимации и замедлять взаимодействие.
  • Не учитывать доступность и игнорировать пользователей с ограничениями по motion.
  • Пытаться анимировать тяжелые свойства и получать рывки интерфейса.
