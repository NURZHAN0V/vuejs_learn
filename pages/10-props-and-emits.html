Раздел 9. Props и события компонентов (emits)

Учимся правильно передавать данные между родителем и ребенком без "магии" и непредсказуемых связей.

Цель урока

Освоить ключевые идеи раздела и закрепить их через короткую пошаговую практику.

Теория простыми словами

1) props: типы, значения по умолчанию, валидация

Props — входные данные компонента. Они описываются явно: тип, default, проверка формата.

2) Однонаправленный поток данных

Данные идут сверху вниз: родитель → ребенок. Это делает поведение интерфейса предсказуемым.

3) emit и defineEmits

Если ребенку нужно сообщить о действии, он отправляет событие через emit, а родитель решает, что делать.

4) Паттерн «родитель владеет состоянием»

Состояние хранится у родителя, ребенок только отображает и отправляет события — так проще поддерживать код.

5) Несколько v-model на одном компоненте

Можно связать несколько полей компонента через именованные модели: v-model:title, v-model:done.

Практика по шагам

Практическое задание: список задач, где дочерний компонент сообщает о действиях родителю.

Мостик: сначала передаем данные вниз через props, затем поднимаем действия вверх через emits — это и есть полный цикл общения компонентов.

Шаг 1. Родитель хранит состояние задач

// TodoList.vue
import { ref } from 'vue'

const todos = ref([
  { id: 1, title: 'Изучить props', done: false },
  { id: 2, title: 'Сделать emits', done: true }
])

Шаг 2. Передай задачу в ребенка через props

<!-- TodoList.vue -->
<TodoItem
  v-for="todo in todos"
  :key="todo.id"
  :todo="todo"
  @toggle="toggleTodo"
  @remove="removeTodo"
/>
// TodoItem.vue
const props = defineProps({
  todo: {
    type: Object,
    required: true
  }
})

Шаг 3. Отправляй события из ребенка через defineEmits

// TodoItem.vue
const emit = defineEmits(['toggle', 'remove'])
<button @click="emit('toggle', props.todo.id)">Сменить статус</button>
<button @click="emit('remove', props.todo.id)">Удалить</button>

Шаг 4. Родитель обрабатывает события

const toggleTodo = (id) => {
  const item = todos.value.find((todo) => todo.id === id)
  if (item) item.done = !item.done
}

const removeTodo = (id) => {
  todos.value = todos.value.filter((todo) => todo.id !== id)
}

Ребенок сообщает "что произошло", а родитель решает "как менять состояние".

Шаг 5. Почему нельзя менять props напрямую

// Плохо: внутри ребенка
props.todo.done = true // так делать не нужно
// Хорошо: отправь событие
emit('toggle', props.todo.id)

Шаг 6. Несколько v-model на одном компоненте

<!-- Parent.vue -->
<TodoEditor
  v-model:title="draftTitle"
  v-model:done="draftDone"
/>
// TodoEditor.vue
const title = defineModel('title')
const done = defineModel('done')

Практическое задание

Сделай список задач с событиями от ребенка к родителю.

  1. Родитель хранит массив задач и методы изменения.
  2. Ребенок получает задачу через props.
  3. Ребенок отправляет toggle и remove через emit.
  4. Родитель обновляет массив по событиям.
  5. Добавь форму редактирования с двумя v-model на одном компоненте.

Мини-практика: рейтинг-виджет, где значение хранится у родителя, а ребенок отправляет событие выбора.

Типичные ошибки

  • Менять props напрямую в дочернем компоненте вместо отправки события.
  • Не описывать события через defineEmits и терять предсказуемость API компонента.
  • Создавать скрытую двустороннюю связь вместо явного паттерна "родитель владеет состоянием".
