Современный учебник Vue.js 3

Раздел 8. Компоненты: создание, регистрация, композиция

Учимся разбивать интерфейс на переиспользуемые части и собирать чистую, поддерживаемую структуру приложения.

Цель урока

Освоить ключевые идеи раздела и закрепить их через короткую пошаговую практику.

Теория простыми словами

1) SFC (.vue) и структура

Один компонент обычно хранится в одном файле .vue и состоит из <template>, <script setup> и <style>.

2) Локальная и глобальная регистрация

Локальная — импорт внутри конкретного родителя. Глобальная — доступно во всем приложении через app.component().

3) Базовые и предметные компоненты

Базовые компоненты — общие (кнопка, карточка). Предметные — для конкретной задачи (карточка профиля, список заказов).

4) Динамические компоненты (<component :is="...">)

Позволяют подставлять нужный компонент на лету, например для табов или переключаемых экранов.

5) Асинхронные компоненты (defineAsyncComponent)

Можно загружать компонент по требованию, чтобы не тянуть весь код сразу и ускорить старт страницы.

Практика по шагам

Практическое задание: собрать страницу профиля из 5–7 компонентов.

Мостик: в каждом шаге мы выделяем один новый компонент и сразу встраиваем его в общую страницу, чтобы структура росла постепенно.

Шаг 1. Создай базовый SFC-компонент

<!-- components/BaseCard.vue -->
<template>
  <section class="rounded-xl border p-4 shadow-sm">
    <slot />
  </section>
</template>

Шаг 2. Локально подключи компонент в странице

<!-- ProfilePage.vue -->
<script setup>
import BaseCard from './components/BaseCard.vue'
</script>

<template>
  <BaseCard>
    <h2>Профиль пользователя</h2>
  </BaseCard>
</template>

Шаг 3. Покажи глобальную регистрацию

// main.js
import { createApp } from 'vue'
import App from './App.vue'
import BaseButton from './components/BaseButton.vue'

const app = createApp(App)
app.component('BaseButton', BaseButton)
app.mount('#app')

После этого <BaseButton /> можно использовать без импорта в каждом файле.

Шаг 4. Собери профиль из предметных компонентов

<!-- ProfilePage.vue -->
<template>
  <ProfileHeader />
  <ProfileContacts />
  <ProfileStats />
  <ProfileSkills />
  <ProfileActivity />
</template>

Так логика и разметка становятся проще в поддержке.

Шаг 5. Динамический компонент для табов

<script setup>
import { ref } from 'vue'
import ProfileAbout from './ProfileAbout.vue'
import ProfileProjects from './ProfileProjects.vue'

const currentTab = ref('about')
const tabMap = {
  about: ProfileAbout,
  projects: ProfileProjects
}
</script>

<template>
  <component :is="tabMap[currentTab]" />
</template>

Шаг 6. Асинхронная загрузка тяжелого блока

import { defineAsyncComponent } from 'vue'

const ProfileCharts = defineAsyncComponent(() =>
  import('./ProfileCharts.vue')
)
<template>
  <ProfileCharts />
</template>

Практическое задание

Собери страницу профиля из 5–7 компонентов.

  1. Выдели общие блоки: карточка, кнопка, секция.
  2. Сделай 5+ предметных компонентов (хедер, контакты, навыки и т.д.).
  3. Добавь один динамический блок через <component :is>.
  4. Загрузи один тяжелый блок асинхронно через defineAsyncComponent.
  5. Сделай аккуратную структуру папок в components/.

Мини-практика: карточки товаров с повторным использованием одного компонента карточки.

Типичные ошибки

  • Создавать "толстые" компоненты с большим количеством несвязанной логики.
  • Выносить в отдельный компонент слишком мелкие фрагменты без реальной пользы.
  • Не переиспользовать базовые компоненты и дублировать одинаковую разметку.
