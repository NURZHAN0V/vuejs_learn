Проект B. Личный кабинет

Собери интерфейс профиля пользователя: карточка, настройки и история действий с фильтрами.

Что реализовать

  • Профиль: имя, email, роль, аватар-заглушка.
  • Редактирование данных через форму с валидацией.
  • Таб «История» с фильтрацией и сортировкой.
  • Сохранение настроек в `localStorage`.

Технические акценты

  • Разбей экран на 3 компонента: `ProfileCard`, `SettingsForm`, `ActivityList`.
  • Применяй `computed` для фильтрации и сортировки истории.
  • Используй `watch` для автосохранения настроек.

Критерий завершения

Пользователь может обновить профиль, увидеть изменения после перезагрузки и быстро найти действия в истории.

Детализация требований

  • Поля профиля: имя, email, телефон, город, роль, короткое bio.
  • Валидация: email обязателен и уникален в пределах mock-данных.
  • История должна поддерживать поиск по тексту и фильтр по типу действия.
  • Действия истории: login, profile_update, password_change, logout.
  • Сохраняй профиль и настройки в localStorage разными ключами.

План работ

  1. Собери layout кабинета и роуты: обзор, настройки, история.
  2. Реализуй форму и слой валидации с сообщениями ошибок.
  3. Добавь список активности с сортировкой по дате.
  4. Подключи авто-сохранение через watch и тест восстановления данных.
