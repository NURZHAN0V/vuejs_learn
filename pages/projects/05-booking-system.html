Проект E. Система бронирования встреч

Продвинутый итог: построить SPA для записи на встречи с календарем, слотами времени и подтверждением брони.

Функции приложения

  • Календарь с доступными слотами на выбранную дату.
  • Создание, перенос и отмена бронирования.
  • Фильтр по специалистам и типу встречи.
  • История бронирований пользователя с поиском.

Техническая реализация

  • Vue Router: отдельные экраны календаря, подтверждения и истории.
  • Pinia для централизованного состояния слотов и бронирований.
  • Работа с API: асинхронная загрузка доступности и отправка заявок.
  • Валидация формы и защита от двойного бронирования одного слота.

Критерий завершения

Пользователь может выбрать дату и слот, забронировать встречу и увидеть запись в истории без ошибок состояния.

Подробные требования

  • Слот: { id, specialistId, date, time, duration, isBooked }.
  • Бронирование: { id, slotId, userName, phone, note, status }.
  • Статусы брони: pending, confirmed, canceled, completed.
  • Блокируй прошедшие даты и пересечение по времени у одного специалиста.
  • Добавь экран администратора для подтверждения/отмены заявок.

Шаги реализации

  1. Собери календарный вид и загрузку доступных слотов из mock API.
  2. Реализуй форму бронирования с валидацией телефона и имени.
  3. Добавь страницу истории и фильтрацию по статусу записи.
  4. Организуй optimistic update с откатом при неуспешном API-ответе.
  5. Покрой ключевые сценарии тестами: booking, cancel, reschedule.
