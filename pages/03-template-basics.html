Раздел 2. База шаблонов: интерполяция и директивы

Учимся выводить данные, управлять HTML, показывать условия и строить списки.

Цель урока

Освоить ключевые идеи раздела и закрепить их через короткую пошаговую практику.

Теория простыми словами

1) Интерполяция {{ }}

Это самый простой способ вывести значение в шаблон: подставляешь переменную в двойные фигурные скобки.

2) v-bind для атрибутов

Когда нужно управлять src, href, class и другими атрибутами из данных, используем v-bind или сокращение :.

3) v-if и v-show

v-if реально добавляет/удаляет блок, v-show только прячет через CSS. Это важно для производительности и логики.

4) v-for и key

Для списка нужен v-for, а для стабильного обновления DOM обязательно добавляй уникальный key.

5) v-once и v-html

v-once рендерит один раз без обновлений. v-html вставляет HTML-строку, но использовать его нужно осторожно из-за XSS-рисков.

Практика по шагам

Практическое задание: сделать карточку профиля с условным блоком и списком.

Мостик: каждый следующий шаг расширяет тот же пример, поэтому после каждого шага у тебя остается рабочий результат, который ты улучшаешь.

Шаг 1. Выведи данные через интерполяцию

<template>
  <h2>{{ name }}</h2>
  <p>{{ role }}</p>
</template>

<script setup>
import { ref } from 'vue'
const name = ref('Анна')
const role = ref('Frontend Developer')
</script>

Это база: сначала просто показываем имя и роль.

Шаг 2. Добавь атрибуты через v-bind

<img :src="avatarUrl" :alt="name" class="h-20 w-20 rounded-full" />
<a :href="profileLink" target="_blank">Профиль</a>

Теперь src/href берутся из данных, а не забиты вручную в HTML.

Шаг 3. Покажи условный блок

<p v-if="isOnline" class="text-emerald-600">Сейчас в сети</p>
<p v-else class="text-slate-500">Не в сети</p>

Если нужно быстро скрывать/показывать без удаления узла, используй v-show.

Шаг 4. Добавь список навыков через v-for и key

<ul>
  <li v-for="skill in skills" :key="skill">
    {{ skill }}
  </li>
</ul>

Без key Vue сложнее корректно обновлять список.

Шаг 5. v-once и v-html на практике

<p v-once>Эта строка не будет обновляться после первого рендера</p>
<div v-html="safeHtml"></div>

v-html применяй только для доверенных данных, иначе можно получить XSS-уязвимость.

Практическое задание

Сделай карточку профиля с условным блоком и списком.

  1. Выведи имя и роль через {{ }}.
  2. Добавь аватар и ссылку через v-bind.
  3. Покажи статус "в сети/не в сети" через v-if / v-else.
  4. Выведи список навыков через v-for + :key.
  5. Добавь строку с v-once и блок с безопасным v-html.

Типичные ошибки

  • Забывать :key в v-for и получать некорректные обновления списка.
  • Использовать v-html для недоверенных данных без очистки.
  • Применять v-if там, где блок часто переключается и лучше подойдет v-show.
