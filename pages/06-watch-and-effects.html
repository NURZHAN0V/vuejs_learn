Раздел 5. Наблюдатели и побочные эффекты

Учимся запускать логику при изменении состояния: сохранять настройки, синхронизировать данные и избегать лишних перерасчетов.

Цель урока

Научиться запускать нужные действия при изменении данных и избегать лишних срабатываний.

Теория простыми словами

1) watch() и его сценарии

watch нужен, когда ты знаешь, за чем следишь: один ref, поле объекта или computed.

2) watchEffect() и авто-зависимости

watchEffect сам собирает зависимости из кода внутри функции. Удобно для быстрых реакций на несколько значений.

3) Очистка эффекта (onCleanup)

Если эффект запускает таймер, подписку или запрос, важно очистить прошлую версию, чтобы не было утечек и гонок.

4) Глубокое наблюдение и его цена

deep: true полезен, но может быть дорогим на больших объектах. Используй только когда действительно нужно.

5) flush и immediate

immediate: true запускает watch сразу. flush управляет моментом выполнения эффекта относительно обновления DOM.

Практика по шагам

Практическое задание: сохранять настройки пользователя в localStorage.

Мостик: на каждом шаге мы добавляем один новый триггер наблюдения, чтобы ты видел разницу между watch и watchEffect в живом примере.

Шаг 1. Состояние настроек

<template>
  <section class="settings-card">
    <h2>Настройки профиля</h2>
    <p>Тема: {{ theme }}</p>
    <p>Язык: {{ settings.language }}</p>
    <p>Уведомления: {{ settings.notifications ? 'Включены' : 'Выключены' }}</p>
  </section>
</template>

<script setup>
import { reactive, ref } from 'vue'

const theme = ref('light')
const settings = reactive({
  language: 'ru',
  notifications: true
})
</script>

Стартовый компонент уже отображает все данные, за которыми будем наблюдать дальше.

Шаг 2. Сохраняй конкретное значение через watch

<template>
  <section class="settings-card">
    <label>Тема</label>
    <select v-model="theme">
      <option value="light">light</option>
      <option value="dark">dark</option>
    </select>
    <p>Текущая тема: {{ theme }}</p>
  </section>
</template>

<script setup>
import { ref, watch } from 'vue'

const theme = ref('light')

watch(theme, (newTheme) => {
  localStorage.setItem('theme', newTheme)
})
</script>

Это классический сценарий: есть один источник и четкая реакция на него.

Шаг 3. Наблюдение за объектом + deep

<template>
  <section class="settings-card">
    <label>Язык</label>
    <select v-model="settings.language">
      <option value="ru">Русский</option>
      <option value="en">English</option>
    </select>

    <label>
      <input v-model="settings.notifications" type="checkbox" />
      Уведомления
    </label>
  </section>
</template>

<script setup>
import { reactive, watch } from 'vue'

const settings = reactive({
  language: 'ru',
  notifications: true
})

watch(
  () => settings,
  (newSettings) => {
    localStorage.setItem('settings', JSON.stringify(newSettings))
  },
  { deep: true }
)
</script>

Используй глубину только при необходимости, иначе будет лишняя нагрузка.

Шаг 4. immediate и flush

<template>
  <section class="settings-card">
    <button @click="theme = 'light'">Light</button>
    <button @click="theme = 'dark'">Dark</button>
    <p>Тема: {{ theme }}</p>
    <p>Лог: {{ watchLog }}</p>
  </section>
</template>

<script setup>
import { ref, watch } from 'vue'

const theme = ref('light')
const watchLog = ref('')

watch(
  theme,
  () => {
    watchLog.value = `Тема обновлена: ${theme.value}`
  },
  {
    immediate: true,
    flush: 'post'
  }
)
</script>

immediate запускает обработчик сразу, flush: 'post' — после обновления DOM.

Шаг 5. watchEffect и очистка эффекта

<template>
  <section class="settings-card">
    <input v-model="query" placeholder="Поиск языка..." />
    <p>Черновик: {{ query }}</p>
    <p>Автосохранено: {{ savedQuery }}</p>
  </section>
</template>

<script setup>
import { ref, watchEffect } from 'vue'

const query = ref('')
const savedQuery = ref('')

watchEffect((onCleanup) => {
  const timerId = setTimeout(() => {
    savedQuery.value = query.value
    localStorage.setItem('search-query', query.value)
  }, 500)

  onCleanup(() => clearTimeout(timerId))
})
</script>

Очистка удаляет старый таймер, если зависимость изменилась до его выполнения.

Шаг 6. Загрузка сохраненных значений при старте

<template>
  <section class="settings-card">
    <p>Тема: {{ theme }}</p>
    <p>Язык: {{ settings.language }}</p>
    <label>
      <input v-model="settings.notifications" type="checkbox" />
      Уведомления
    </label>
  </section>
</template>

<script setup>
import { reactive, ref, watch } from 'vue'

const theme = ref('light')
const settings = reactive({
  language: 'ru',
  notifications: true
})

const savedTheme = localStorage.getItem('theme')
if (savedTheme) theme.value = savedTheme

const savedSettings = localStorage.getItem('settings')
if (savedSettings) {
  Object.assign(settings, JSON.parse(savedSettings))
}

watch(theme, (newTheme) => localStorage.setItem('theme', newTheme), { immediate: true })
watch(
  () => settings,
  (newSettings) => localStorage.setItem('settings', JSON.stringify(newSettings)),
  { deep: true, immediate: true }
)
</script>

Финальный шаг: настройки загружаются при старте и автоматически сохраняются при каждом изменении.

Практическое задание

Сохраняй настройки пользователя в localStorage.

  1. Добавь поля: тема, язык, уведомления.
  2. Сохраняй тему через watch(theme, ...).
  3. Сохраняй объект настроек с deep: true.
  4. Сделай автозагрузку настроек при старте.
  5. Добавь watchEffect с очисткой таймера через onCleanup.

Мини-практика: автосохранение текста черновика в localStorage.

Типичные ошибки

  • Ставить deep: true без необходимости и перегружать реактивность.
  • Менять источник внутри watch без контроля и запускать бесконечный цикл.
  • Забывать очистку в watchEffect при таймерах и подписках.
