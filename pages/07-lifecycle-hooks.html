Раздел 6. Жизненный цикл компонента

Разберем, в какой момент запускать код: при создании, обновлении и удалении компонента.

Цель урока

Научиться выполнять код в правильный момент жизни компонента и корректно очищать ресурсы.

Теория простыми словами

1) onMounted, onUpdated, onUnmounted

onMounted — компонент уже в DOM. onUpdated — компонент обновился. onUnmounted — компонент удаляется, здесь нужно чистить таймеры и подписки.

2) onBeforeMount, onBeforeUpdate, onBeforeUnmount

Эти хуки срабатывают "до" действия. Полезны, когда важно выполнить логику до монтирования/обновления/удаления.

3) onErrorCaptured

Позволяет поймать ошибку дочернего компонента и не "уронить" весь экран. Подходит для безопасных fallback-сценариев.

4) nextTick() в реальных задачах

Если нужно работать с DOM сразу после обновления данных, используй nextTick().

Практика по шагам

Практическое задание: таймер с правильной очисткой при удалении компонента.

Мостик: каждый шаг показывает новый этап жизни компонента, чтобы ты понимал не только "что делать", но и "когда делать".

Шаг 1. Подготовь состояние таймера

import { ref } from 'vue'

const seconds = ref(0)
let timerId = null

Шаг 2. Запускай таймер в onMounted

import { onMounted } from 'vue'

onMounted(() => {
  timerId = setInterval(() => {
    seconds.value += 1
  }, 1000)
})

Шаг 3. Очисти таймер в onUnmounted

import { onUnmounted } from 'vue'

onUnmounted(() => {
  clearInterval(timerId)
})

Это защищает от утечки памяти и "фоновой" работы удаленного компонента.

Шаг 4. onBeforeUpdate и onUpdated для отладки

import { onBeforeUpdate, onUpdated } from 'vue'

onBeforeUpdate(() => {
  console.log('Перед обновлением DOM')
})

onUpdated(() => {
  console.log('После обновления DOM')
})

Шаг 5. Используй nextTick после изменения данных

import { nextTick } from 'vue'

const resetTimer = async () => {
  seconds.value = 0
  await nextTick()
  console.log('DOM уже обновился после сброса')
}

Шаг 6. Перехват ошибок через onErrorCaptured

import { onErrorCaptured } from 'vue'

onErrorCaptured((error, instance, info) => {
  console.error('Ошибка в дочернем компоненте:', error, info)
  return false
})

Так можно централизованно фиксировать ошибки в сложном интерфейсе.

Практическое задание

Собери таймер с корректным жизненным циклом.

  1. Запускай счетчик в onMounted.
  2. Очищай интервал в onUnmounted.
  3. Добавь кнопку сброса и используй nextTick.
  4. Выведи лог в onBeforeUpdate и onUpdated.
  5. Поймай тестовую ошибку через onErrorCaptured.

Мини-практика: подключи обработчик resize в onMounted и удаляй его в onUnmounted.

Типичные ошибки

  • Создавать таймеры и слушатели событий без очистки в onUnmounted.
  • Обращаться к DOM до onMounted, когда элемент еще не создан.
  • Запускать тяжелую логику в каждом обновлении и тормозить интерфейс.
