Раздел 20. TypeScript в Vue 3

Учимся добавлять типизацию в Vue-компоненты, composables, stores и router без лишней сложности.

Цель урока

Освоить ключевые идеи раздела и закрепить их через короткую пошаговую практику.

Теория простыми словами

1) База TS в Vue без перегруза

Главная цель — не "типизировать всё подряд", а защитить критичные места: данные, контракты и API.

2) Типы для ref, reactive, computed

Типизация состояния сразу показывает ошибки и подсказывает структуру данных в IDE.

3) Типизация props/emits/slots

Контракты компонента становятся явными и безопасными, особенно в командной разработке.

4) Типы API-ответов, Pinia, Router

Типы запросов и маршрутов снижают количество ошибок "в рантайме" и ускоряют рефакторинг.

5) Полезные шаблоны типизации

Используй интерфейсы/типы для доменных сущностей и не злоупотребляй any.

Практика по шагам

Практическое задание: типизированный модуль «Каталог товаров».

Мостик: типы вводятся по слоям — от state к API и компонентам, чтобы ты сразу видел, где они экономят время и ошибки.

Шаг 1. Опиши доменные типы

export interface Product {
  id: number
  title: string
  price: number
  inStock: boolean
}

export interface ProductsResponse {
  items: Product[]
  total: number
}

Шаг 2. Типизируй ref/reactive/computed

import { ref, reactive, computed } from 'vue'
import type { Product } from './types'

const products = ref([])
const filters = reactive({ minPrice: 0, onlyInStock: false })
const visibleProducts = computed(() =>
  products.value.filter(p => p.price >= filters.minPrice && (!filters.onlyInStock || p.inStock))
)

Шаг 3. Типизируй props/emits/slots

<script setup lang="ts">
import type { Product } from '../types'

const props = defineProps<{ product: Product }>()
const emit = defineEmits<{ (e: 'add-to-cart', id: number): void }>()
defineSlots<{ default(props: { product: Product }): any }>()
</script>

Шаг 4. Тип API-ответа и fetch

import type { ProductsResponse } from './types'

async function loadProducts(): Promise {
  const res = await fetch('/api/products')
  const data: ProductsResponse = await res.json()
  products.value = data.items
}

Шаг 5. Типизация Pinia store

import { defineStore } from 'pinia'
import { ref } from 'vue'
import type { Product } from './types'

export const useCatalogStore = defineStore('catalog', () => {
  const items = ref([])
  function setItems(next: Product[]) {
    items.value = next
  }
  return { items, setItems }
})

Шаг 6. Типизация Router meta

// router.ts
const routes = [
  {
    path: '/catalog',
    component: () => import('./pages/CatalogPage.vue'),
    meta: { requiresAuth: false as boolean }
  }
]

Шаг 7. Шаблоны, которые реально помогают

type Id = number
type Nullable<T> = T | null
type ApiState<T> = {
  data: Nullable<T>
  loading: boolean;
  error: Nullable<string>
}

Практическое задание

Собери типизированный модуль «Каталог товаров».

  1. Опиши типы Product и ProductsResponse.
  2. Типизируй состояние, фильтры и computed.
  3. Типизируй карточку товара через defineProps/defineEmits.
  4. Сделай типизированный store каталога.
  5. Проверь, что в коде нет any без явной причины.

Мини-практика: безопасная форма с подсказками типов и проверкой полей.

Типичные ошибки

  • Ставить any вместо точных типов и терять пользу от TypeScript.
  • Типизировать все подряд слишком рано, перегружая учебный код.
  • Не описывать типы API-ответов и получать ошибки в рантайме.
