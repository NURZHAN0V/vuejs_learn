Раздел 16. Управление состоянием с Pinia

Учимся хранить общее состояние централизованно: stores, actions, getters, модули и persist.

Цель урока

Освоить ключевые идеи раздела и закрепить их через короткую пошаговую практику.

Теория простыми словами

1) Почему локального state иногда недостаточно

Когда одни и те же данные нужны в разных страницах/компонентах, локальный state начинает дублироваться и расходиться.

2) Setup Store и Option Store

Pinia поддерживает два стиля: функциональный (Setup) и объектный (Option). Оба рабочие, выбирай удобный для команды.

3) State, Getters, Actions

State — данные, Getters — производные значения, Actions — изменения и бизнес-логика.

4) Разделение на store domains

Лучше иметь несколько store по зонам: auth, cart, products, чем один огромный store.

5) Плагины Pinia и Persist

Плагинами расширяют поведение store. Persist сохраняет состояние между перезагрузками.

Практика по шагам

Практическое задание: корзина интернет-магазина на Pinia.

Мостик: последовательно строим store — state, getters, actions и интеграция в UI, чтобы видеть полный цикл состояния в приложении.

Шаг 1. Подключи Pinia

// main.js
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import App from './App.vue'

const app = createApp(App)
app.use(createPinia())
app.mount('#app')

Шаг 2. Setup Store для корзины

// stores/cart.js
import { defineStore } from 'pinia'
import { computed, ref } from 'vue'

export const useCartStore = defineStore('cart', () => {
  const items = ref([])
  const total = computed(() => items.value.reduce((sum, i) => sum + i.price * i.qty, 0))

  function addItem(product) {
    const found = items.value.find(i => i.id === product.id)
    if (found) found.qty += 1
    else items.value.push({ ...product, qty: 1 })
  }

  function removeItem(id) {
    items.value = items.value.filter(i => i.id !== id)
  }

  return { items, total, addItem, removeItem }
})

Шаг 3. Option Store (альтернатива)

export const useCartOptionStore = defineStore('cartOption', {
  state: () => ({ items: [] }),
  getters: {
    total: (state) => state.items.reduce((sum, i) => sum + i.price * i.qty, 0)
  },
  actions: {
    addItem(product) {
      const found = this.items.find(i => i.id === product.id)
      if (found) found.qty += 1
      else this.items.push({ ...product, qty: 1 })
    }
  }
})

Шаг 4. Используй store в компоненте

<script setup>
import { useCartStore } from '../stores/cart'
const cart = useCartStore()
</script>

<template>
  <p>Итого: {{ cart.total }} ₽</p>
  <button @click="cart.addItem({ id: 1, title: 'Книга', price: 900 })">Добавить</button>
</template>

Шаг 6. Persist состояния (практика)

// stores/cart.js
import { watch } from 'vue'

// внутри store
const raw = localStorage.getItem('cart-items')
if (raw) items.value = JSON.parse(raw)

watch(items, (val) => {
  localStorage.setItem('cart-items', JSON.stringify(val))
}, { deep: true })

Шаг 7. Лучшие практики структуры

- Держи store маленькими и предметными
- Не храни временный UI-state глобально без нужды
- Выноси бизнес-правила в actions
- Не дублируй одни и те же данные в разных store

Практическое задание

Собери корзину интернет-магазина на Pinia.

  1. Создай store корзины с items, total, add/remove.
  2. Покажи количество товаров и итог в шапке.
  3. Сделай add/remove из разных компонентов (список + мини-корзина).
  4. Добавь persist в localStorage.
  5. Проверь, что после перезагрузки корзина восстанавливается.

Мини-практика: auth-store с ролями (guest/user/admin) и getter'ом доступа.

Типичные ошибки

  • Хранить в store временное UI-состояние, которое должно жить только в одном компоненте.
  • Дублировать одни и те же данные в нескольких store.
  • Изменять state из произвольных мест без явных actions.
