Раздел 19. <script setup> и современные макросы

Учимся писать современный чистый код компонентов через макросы Vue 3 и <script setup>.

Цель урока

Научиться писать более чистые и короткие компоненты с современным подходом к описанию логики.

Теория простыми словами

1) <script setup> основы

Меньше шаблонного кода, переменные и функции сразу доступны в шаблоне без return.

2) defineProps, defineEmits

Явно описывают входные данные и события компонента, делая контракт компонента прозрачным.

3) defineExpose, defineOptions

defineExpose открывает часть API наружу, defineOptions задает опции компонента в новом стиле.

4) defineModel и defineSlots

defineModel упрощает v-model в дочерних компонентах, defineSlots — типизирует и описывает слоты.

5) Авто-импорт и читаемость

Следи за читаемостью: даже с авто-импортами не перегружай компонент логикой и держи ясную структуру.

Практика по шагам

Практическое задание: переписать набор компонентов на <script setup>.

Мостик: мы переносим один и тот же компонент по шагам, чтобы увидеть, как макросы упрощают код без потери прозрачности.

Шаг 1. Базовый переход на script setup

<script setup>
import { ref } from 'vue'
const count = ref(0)
const increment = () => count.value++
</script>

<template>
  <button @click="increment">{{ count }}</button>
</template>

Шаг 2. defineProps и defineEmits

<script setup>
const props = defineProps({
  title: { type: String, required: true }
})
const emit = defineEmits(['save'])
</script>

<template>
  <h3>{{ props.title }}</h3>
  <button @click="emit('save')">Сохранить</button>
</template>

Шаг 3. defineExpose и defineOptions

<script setup>
import { ref } from 'vue'
defineOptions({ name: 'ProfileEditor' })
const resetCount = ref(0)
const reset = () => { resetCount.value++ }
defineExpose({ reset })
</script>

Родитель сможет вызвать метод reset() через ref к дочернему компоненту.

Шаг 4. defineModel для v-model

<script setup>
const model = defineModel()
</script>

<template>
  <input v-model="model" />
</template>
<!-- Parent -->
<SmartInput v-model="form.name" />

Шаг 5. Несколько моделей в одном компоненте

<script setup>
const title = defineModel('title')
const done = defineModel('done')
</script>
<TodoEditor v-model:title="draftTitle" v-model:done="draftDone" />

Шаг 6. defineSlots и читаемая структура

<script setup>
defineSlots()
</script>

<template>
  <header><slot name="header" /></header>
  <main><slot /></main>
</template>

Практическое задание

Перепиши набор компонентов на <script setup>.

  1. Переведи 2–3 старых компонента на script setup.
  2. Явно опиши props и emits.
  3. Добавь компонент с defineModel.
  4. Сделай компонент со слотами и defineSlots.
  5. Проверь, что код стал короче и понятнее.

Мини-практика: компонент формы с несколькими моделями.

Типичные ошибки

  • Смешивать старый и новый стиль без причин и ухудшать читаемость.
  • Использовать макросы без явного понимания, зачем они нужны в компоненте.
  • Забывать про границы API компонента при defineExpose и defineModel.
