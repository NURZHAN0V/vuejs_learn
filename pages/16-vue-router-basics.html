Раздел 15. Маршрутизация с Vue Router

Учимся строить многостраничные SPA: маршруты, вложенность, guards, защищенные страницы и удобную навигацию.

Цель урока

Научиться собирать базовую навигацию приложения: создавать маршруты, делать переходы между страницами и добавлять защиту для закрытых разделов.

Теория простыми словами

1) Создание роутера

Роутер описывает, какой компонент должен открываться по какому URL.

2) RouterLink, RouterView

RouterLink — ссылки внутри SPA, RouterView — место, где рендерится текущая страница.

3) Динамические и вложенные маршруты

Маршруты с параметрами (/posts/:id) и дочерние страницы для секций личного кабинета.

4) Редиректы, 404, guards и метаданные

Можно перенаправлять, обрабатывать несуществующие пути и защищать страницы через guards и meta.

5) Lazy loading и scroll behavior

Страницы грузятся по требованию, а поведение прокрутки можно настроить для удобства пользователя.

Практика по шагам

Практическое задание: личный кабинет с публичными и защищенными страницами.

Мостик: на каждом шаге добавляется один слой маршрутизации — сначала базовые страницы, затем параметры, а после этого защита и fallback.

Шаг 1. Создай router/index.js

import { createRouter, createWebHistory } from 'vue-router'
import HomePage from '../pages/HomePage.vue'

const routes = [
  { path: '/', name: 'home', component: HomePage }
]

export const router = createRouter({
  history: createWebHistory(),
  routes
})

Шаг 2. Подключи RouterLink и RouterView

<template>
  <nav>
    <RouterLink to="/">Главная</RouterLink>
    <RouterLink to="/dashboard">Кабинет</RouterLink>
  </nav>
  <RouterView />
</template>

Шаг 3. Добавь динамический маршрут

{ path: '/posts/:id', name: 'post-details', component: () => import('../pages/PostDetails.vue') }
import { useRoute } from 'vue-router'
const route = useRoute()
const postId = route.params.id

Шаг 4. Вложенные маршруты для кабинета

{
  path: '/dashboard',
  component: () => import('../pages/DashboardLayout.vue'),
  children: [
    { path: '', name: 'dashboard-home', component: () => import('../pages/DashboardHome.vue') },
    { path: 'settings', name: 'dashboard-settings', component: () => import('../pages/DashboardSettings.vue') }
  ]
}

Шаг 5. Редирект и 404

{ path: '/profile', redirect: '/dashboard' },
{ path: '/:pathMatch(.*)*', name: 'not-found', component: () => import('../pages/NotFound.vue') }

Шаг 6. Глобальный guard с meta

{
  path: '/dashboard',
  component: () => import('../pages/DashboardLayout.vue'),
  meta: { requiresAuth: true }
}
router.beforeEach((to) => {
  const isAuthenticated = Boolean(localStorage.getItem('token'))
  if (to.meta.requiresAuth && !isAuthenticated) {
    return { name: 'home' }
  }
})

Шаг 7. Локальный guard и scroll behavior

{
  path: '/danger-zone',
  component: () => import('../pages/DangerZone.vue'),
  beforeEnter: (to, from) => {
    const confirmed = window.confirm('Перейти в опасную зону?')
    return confirmed || false
  }
}
const router = createRouter({
  history: createWebHistory(),
  routes,
  scrollBehavior(to, from, savedPosition) {
    if (savedPosition) return savedPosition
    return { top: 0 }
  }
})

Практическое задание

Собери личный кабинет с публичными и защищенными страницами.

  1. Создай публичные страницы: главная, логин, 404.
  2. Создай защищенный /dashboard с вложенными роутами.
  3. Добавь динамический маршрут /posts/:id.
  4. Сделай глобальный beforeEach с проверкой meta.requiresAuth.
  5. Добавь lazy loading и scroll behavior.

Мини-практика: блог с маршрутом /posts/:id и загрузкой поста по id.

Типичные ошибки

  • Не обрабатывать маршрут 404 и оставлять пользователя в тупике.
  • Смешивать логику прав доступа в компонентах вместо guard в роутере.
  • Забывать про динамические параметры и проверку их корректности.
