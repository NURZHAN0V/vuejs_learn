Современный учебник Vue.js 3

Раздел 1. Введение в Vue.js 3 и запуск первого проекта

Короткий и практический старт: что такое Vue, когда его выбирать, как запустить проект и быстро изменить стартовый экран.

Цель урока

Понять базовые принципы Vue и уверенно запустить первый проект на Vite с понятной структурой файлов.

Теория простыми словами (что внутри)

1) Что такое Vue простыми словами (реактивный UI)

Vue можно представить как "умный слой" между данными и экраном. Меняешь данные — интерфейс обновляется сам. Это и есть реактивность.

2) Когда Vue подходит лучше всего

  • Когда нужен понятный вход для новичка.
  • Когда проект растет постепенно: от маленького виджета до большого SPA.
  • Когда важно быстро делать интерфейсы и не тонуть в сложности.

3) Варианты подключения: CDN и Vite

  • CDN: удобно для быстрого эксперимента в одном HTML-файле.
  • Vite: стандарт для реальных проектов (быстрый запуск, удобная сборка, структура).

4) Как все связано: фреймворк, Node.js, фронтенд и бэкенд

  • Фреймворк — это "каркас" приложения с готовыми правилами и инструментами. Vue дает структуру для интерфейса.
  • Node.js нужен не для показа сайта пользователю, а для разработки: установить пакеты, запустить Vite, собрать проект.
  • Фронтенд-сервер (Vite dev server) отдает HTML/CSS/JS в браузер во время разработки.
  • Бэкенд-сервер отдает данные и бизнес-логику (например, список задач, авторизация, база данных).
  • Связка обычно такая: Браузер → фронтенд (Vue) → API бэкенда → база данных.

5) Структура проекта: main.js, App.vue, components

  • main.js — точка входа. Здесь приложение подключается к странице.
  • App.vue — корневой компонент, "главный экран".
  • components/ — папка для переиспользуемых частей интерфейса.

6) DevTools для Vue

DevTools помогает видеть компоненты, их состояние и события. Это как "рентген" для интерфейса: сразу видно, что именно происходит.

Практика по шагам

Цель практики: запустить проект на Vite и поменять стартовый экран.

Перед стартом: что должно быть установлено

  • Node.js 20+ (проверка: node -v).
  • npm (проверка: npm -v).

Если команда не найдена, сначала установи Node.js с официального сайта, затем открой новый PowerShell.

Как создать Vue-приложение с нуля (Windows PowerShell)

Скопируй и выполни команды по порядку:

# 1) Перейди в папку, где будет проект
cd C:\Git

# 2) Создай новый Vue-проект
npm create vue@latest my-first-vue-app

# 3) Перейди в проект
cd .\my-first-vue-app

# 4) Установи зависимости (npm install или npm i)
npm install

# 5) Запусти проект
npm run dev

После запуска в терминале появится локальный адрес (обычно http://localhost:5173). Открой его в браузере.

Примечание: npm install и npm i — это одно и то же.

Где здесь установка Vue

  • Команда npm create vue@latest ... запускает мастер создания проекта Vue.
  • Vue ставится не "глобально на компьютер", а в папку конкретного проекта.
  • Команда npm install устанавливает все нужные зависимости в этом проекте.
  • Итог: ты уже установил Vue правильно, если проект создался и запускается через npm run dev.

Шаг 1. Создай проект

Зачем нужен шаг: создаем "скелет" приложения с готовой структурой файлов.

npm create vue@latest my-first-vue-app

Что изменится: появится папка проекта со стартовыми файлами Vue.

Шаг 2. Установи зависимости и запусти проект

Зачем нужен шаг: без зависимостей и dev-сервера приложение не откроется в браузере.

cd my-first-vue-app
npm install
npm run dev

Связь с шагом 1: теперь "скелет" превращается в рабочее приложение.

Шаг 3. Найди точку входа и главный компонент

Зачем нужен шаг: важно понять, откуда приложение стартует.

Фрагмент 1 — src/main.js:

import { createApp } from 'vue'
import App from './App.vue'

createApp(App).mount('#app')

Что делает фрагмент: создает приложение из App.vue и вставляет его в элемент с id app.

Шаг 4. Измени стартовый экран

Зачем нужен шаг: проверяем, что проект работает и вы контролируете вывод на странице.

Фрагмент 2 — минимальный src/App.vue:

<template>
  <main>
    <h1>Привет, Vue!</h1>
    <p>Мой первый проект успешно запущен.</p>
  </main>
</template>

Связь с предыдущим шагом: main.js загружает App.vue, поэтому изменения видны сразу.

Шаг 5. Подключи Vue DevTools

Зачем нужен шаг: видеть дерево компонентов и состояние приложения в реальном времени. После установки расширения открой вкладку Vue в инструментах разработчика браузера.

Практическое задание

Создать «Привет, Vue».

  1. Создай и запусти проект командами из блока «Как создать Vue-приложение с нуля».
  2. Открой src/App.vue.
  3. Сделай заголовок «Привет, Vue!» и добавь короткое описание о себе.
  4. Добавь текущую дату обычным текстом.
  5. Проверь страницу в браузере и убедись, что все отображается.

Типичные ошибки

  • Запускать команды не в папке проекта и получать "package.json not found".
  • Путать точку входа src/main.js и корневой компонент App.vue.
  • Использовать CDN-подход там, где нужен полноценный проект на Vite.
