Раздел 11. Composition API и composables

Учимся выносить и переиспользовать логику так, чтобы код оставался чистым и масштабируемым.

Цель урока

Научиться выносить повторяющуюся логику в отдельные части и использовать ее в разных компонентах.

Теория простыми словами

1) Логика внутри setup

В Composition API вся логика компонента собирается в setup (или <script setup>) и становится проще для повторного использования.

2) Что такое composable и правила именования

Composable — это функция с логикой, обычно с именем useSomething, например: useFetch, useToggle.

3) Передача параметров и возврат API

Composable принимает входные параметры и возвращает нужные данные/методы наружу: state, computed, actions.

4) Организация composables/

Обычно создают папку composables/, где каждая функция отвечает за одну понятную задачу.

5) Антипаттерны в composables

Опасные места: скрытые side effects, глобальные мутации без контроля, неочищенные таймеры и подписки.

Практика по шагам

Практическое задание: useFetch, useToggle, useLocalStorage.

Мостик: каждый новый composable решает отдельную задачу, а в конце ты объединяешь их в один понятный рабочий сценарий.

Шаг 1. Базовый composable useToggle

// composables/useToggle.js
import { ref } from 'vue'

export function useToggle(initial = false) {
  const state = ref(initial)
  const toggle = () => {
    state.value = !state.value
  }
  return { state, toggle }
}
<!-- SettingsPanel.vue -->
<script setup>
import { useToggle } from './composables/useToggle'

const { state: isOpen, toggle } = useToggle(false)
</script>

<template>
  <button @click="toggle">{{ isOpen ? 'Скрыть' : 'Показать' }} настройки</button>
</template>

Сразу видно практику: composable инкапсулирует состояние и действие, компонент только использует API.

Шаг 2. useLocalStorage с параметрами

// composables/useLocalStorage.js
import { ref, watch } from 'vue'

export function useLocalStorage(key, initialValue) {
  const stored = localStorage.getItem(key)
  const value = ref(stored ? JSON.parse(stored) : initialValue)

  watch(value, (newValue) => {
    localStorage.setItem(key, JSON.stringify(newValue))
  }, { deep: true })

  return value
}
<!-- ThemeSwitcher.vue -->
<script setup>
import { useLocalStorage } from './composables/useLocalStorage'

const theme = useLocalStorage('theme', 'light')
</script>

<template>
  <select v-model="theme">
    <option value="light">Light</option>
    <option value="dark">Dark</option>
  </select>
</template>

Тема переживает перезагрузку страницы без ручной логики в компоненте.

Шаг 3. useFetch для загрузки данных

// composables/useFetch.js
import { ref } from 'vue'

export function useFetch(url) {
  const data = ref(null)
  const loading = ref(false)
  const error = ref(null)

  const execute = async () => {
    loading.value = true
    error.value = null
    try {
      const res = await fetch(url)
      data.value = await res.json()
    } catch (e) {
      error.value = e
    } finally {
      loading.value = false
    }
  }

  return { data, loading, error, execute }
}
<!-- UsersList.vue -->
<script setup>
import { onMounted } from 'vue'
import { useFetch } from './composables/useFetch'

const { data, loading, error, execute } = useFetch('https://jsonplaceholder.typicode.com/users')
onMounted(execute)
</script>

<template>
  <p v-if="loading">Загрузка...</p>
  <p v-else-if="error">Ошибка: {{ error.message }}</p>
  <ul v-else>
    <li v-for="user in data" :key="user.id">{{ user.name }}</li>
  </ul>
</template>

Компонент получает готовые состояния запроса и не дублирует boilerplate-код.

Шаг 4. Использование composables в компоненте

<script setup>
import { onMounted } from 'vue'
import { useToggle } from './composables/useToggle'
import { useFetch } from './composables/useFetch'
import { useLocalStorage } from './composables/useLocalStorage'

const { state: isOpen, toggle } = useToggle(false)
const { data, loading, execute } = useFetch('https://jsonplaceholder.typicode.com/users')
const theme = useLocalStorage('theme', 'light')

onMounted(execute)
</script>

<template>
  <button @click="toggle">{{ isOpen ? 'Скрыть' : 'Показать' }} панель</button>
  <p>Тема: {{ theme }}</p>
  <p v-if="loading">Загрузка пользователей...</p>
  <p v-else>Пользователей: {{ data?.length || 0 }}</p>
</template>

Несколько composable-функций объединяются в одном компоненте без смешивания ответственности.

Шаг 5. Папка composables и чистая структура

src/
  composables/
    useFetch.js
    useToggle.js
    useLocalStorage.js
  components/
  pages/

Одна функция — одна зона ответственности. Это упрощает поддержку и тестирование.

Шаг 6. Антипаттерн и исправление

// Плохо: скрытый side effect при импорте
setInterval(() => console.log('тик'), 1000)
// Хорошо: запуск/очистка контролируются явными методами
export function useTicker() {
  let id = null
  const start = () => { id = setInterval(() => console.log('тик'), 1000) }
  const stop = () => clearInterval(id)
  return { start, stop }
}
<!-- TimerWidget.vue -->
<script setup>
import { onMounted, onUnmounted } from 'vue'
import { useTicker } from './composables/useTicker'

const { start, stop } = useTicker()
onMounted(start)
onUnmounted(stop)
</script>

У эффекта есть явный жизненный цикл: запуск и очистка контролируются компонентом.

Практическое задание

Реализуй три composable: useFetch, useToggle, useLocalStorage.

  1. Сделай useToggle с параметром начального значения.
  2. Сделай useLocalStorage с ключом и default-значением.
  3. Сделай useFetch с состояниями loading/error/data.
  4. Подключи их в одном компоненте страницы.
  5. Убедись, что нет скрытых побочных эффектов при импорте.

Мини-практика: useFormValidation для проверки обязательных полей формы.

Типичные ошибки

  • Скрывать в composable побочные эффекты без явной очистки и контроля.
  • Возвращать из composable слишком широкий API, который трудно поддерживать.
  • Смешивать независимые задачи в одном composable вместо разделения по ответственности.
