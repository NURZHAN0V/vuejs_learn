Раздел 10. Слоты: гибкие и переиспользуемые компоненты

Учимся строить компоненты-конструкторы, где внешний шаблон сам управляет содержимым внутри обертки.

Цель урока

Научиться делать гибкие компоненты-обертки, чтобы легко менять их содержимое под задачу.

Теория простыми словами

1) Обычный слот

Слот — это "окно" внутри компонента, куда родитель может вставить произвольный контент.

2) Именованные слоты

Когда у компонента несколько зон (например: заголовок, тело, футер), используются именованные слоты.

3) Scoped slots

Ребенок может передавать данные в шаблон родителя через slot props, и родитель сам решает, как их отрисовать.

4) Фолбэк-содержимое

Если родитель ничего не передал в слот, компонент может показать дефолтный контент.

Практика по шагам

Практическое задание: универсальный Modal и Card через слоты.

Мостик: двигаемся от простого слота к именованным и scoped, чтобы на каждом шаге расширять гибкость одного и того же компонента.

Шаг 1. Обычный слот в Card

<!-- BaseCard.vue -->
<template>
  <section class="rounded-xl border p-4">
    <slot />
  </section>
</template>
<!-- Parent.vue -->
<BaseCard>
  <h3>Карточка пользователя</h3>
  <p>Контент передан через слот</p>
</BaseCard>

Шаг 2. Именованные слоты в Modal

<!-- BaseModal.vue -->
<template>
  <div class="rounded-xl bg-white p-4 shadow">
    <header class="mb-3"><slot name="header" /></header>
    <main class="mb-3"><slot /></main>
    <footer><slot name="footer" /></footer>
  </div>
</template>
<BaseModal>
  <template #header>Подтверждение</template>
  Вы точно хотите удалить запись?
  <template #footer>
    <button>Отмена</button>
    <button>Удалить</button>
  </template>
</BaseModal>

Шаг 3. Scoped slot: ребенок передает данные

<!-- UserInfo.vue -->
<template>
  <slot :user="user" :isOnline="isOnline" />
</template>

<script setup>
const user = { name: 'Анна', age: 24 }
const isOnline = true
</script>
<UserInfo v-slot="{ user, isOnline }">
  <p>{{ user.name }} ({{ user.age }})</p>
  <p>Статус: {{ isOnline ? 'online' : 'offline' }}</p>
</UserInfo>

Шаг 4. Фолбэк-контент по умолчанию

<!-- EmptyStateCard.vue -->
<template>
  <div class="rounded-xl border p-4">
    <slot>Нет данных для отображения</slot>
  </div>
</template>

Если слот не передан, компонент покажет сообщение по умолчанию.

Шаг 5. Комбинация Card + Modal

<BaseCard>
  <template #default>
    <h3>Профиль</h3>
    <button @click="isModalOpen = true">Открыть детали</button>
  </template>
</BaseCard>

<BaseModal v-if="isModalOpen">
  <template #header>Детали профиля</template>
  Дополнительная информация
  <template #footer>
    <button @click="isModalOpen = false">Закрыть</button>
  </template>
</BaseModal>

Практическое задание

Собери универсальные Modal и Card через слоты.

  1. Сделай базовый Card с обычным слотом.
  2. Собери Modal с именованными слотами (header/footer).
  3. Добавь fallback-содержимое в один из слотов.
  4. Сделай scoped slot для передачи данных в шаблон родителя.
  5. Проверь, что компоненты переиспользуются в 2+ местах.

Мини-практика: таблица с кастомными ячейками через scoped slots.

Типичные ошибки

  • Перегружать компонент множеством слотов без четкой структуры API.
  • Использовать слот там, где достаточно простого prop.
  • Не задавать fallback-содержимое для важных областей интерфейса.
